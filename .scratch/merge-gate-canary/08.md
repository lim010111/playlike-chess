## Injected defect

- **File touched**: `src/engine/playlike_engine/random_move.py`
- **Line(s)**: `import secrets` added; module-level `random`
  swapped to `secrets` in the `chooser` ternary.
- **What changed**:
  1. `import secrets` (new line in the import block).
  2. `chooser = rng if rng is not None else random` →
     `chooser = rng if rng is not None else secrets`.
- **Why it might be a defect (Codex view)**: `secrets` documents
  itself as "for security-sensitive applications" (tokens,
  passwords). Using it for game-move random looks like
  cargo-culted defensive coding. Expected severity: `low` or
  `medium` (overengineering, not a real bug).

## Dismiss/unsure grounds

- Both `random.choice` and `secrets.choice` guarantee uniform
  selection. There is no behavioural regression.
- Project docs (`AGENTS.md`, `src/engine/CONTEXT.md`,
  `src/web/CONTEXT.md`, PRD) do not prescribe a random source for
  engine moves. The validator has no project-internal citation to
  dismiss **with intent** *or* uphold **with intent**.
- Per 04.md's structural reasoning: γ Strict's
  "no dismiss-citation found" default is `uphold`, not `unsure`.
  Reaching `unsure` requires the agent to declare an unresolvable
  ambiguity — orthogonal to citation availability.

**Combined #23 AC #4 evidence:**

The `validator-unsure` path requires both (1) Codex to produce a
finding and (2) validator to find both-sides-equally-citable. Each
attempt failed at a *different step* of this chain:

- Make the injection clearly defect-shaped → Codex flags it → but
  the project's own documented contract supplies *uphold-side*
  citation (04.md).
- Make the injection genuinely ambiguous (no doc opinion either
  way) → Codex doesn't bother flagging it → the validator path
  never engages (this run).

The middle case "Codex flags it, validator genuinely sits on the
fence" requires an injection that (a) Codex grades but (b) the
project documents have *no opinion on* — but the kind of issue
the project documents don't cover is exactly the kind Codex tends
to either not flag (as here) or downgrade. The path is reachable
in principle, but reaching it via synthetic injection at the
single-PR scale is structurally unfavoured by the same calibration
that makes the gate useful.

**Closure decision (#23 AC #4):** 2 attempts of evidence, two
distinct miss-patterns, both consistent with 04.md's structural
γ Strict argument → close C4 in Dimension A as
*reachable-in-principle, not reachable via single-PR synthetic
injection under current calibration*. The path is not a tooling
bug; it is calibration-favoured rarity.
